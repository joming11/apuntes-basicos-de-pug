Templates

usar templates es util para modularizar el codigo

se divide el codigo en secciones que al final se compilan en un solo archivo

el archivo final simplemente debe tener las extensiones hacia los otros archivos .pug

extends + la ruta de carpetas/nombredelarchivo sin la extension

no se pueden agregar codigo despues de agregar un extends, solo se puede agregar block o mixin de pug

Bloques

en el archivo donde esta el codigo, del cual hacemos el extends, alli se debe se debe colocar block y el nombre del block, por ejemplo, block main

en el archivo que estamos extendiendo usamos block main y alli se crea el codgio que queramos añadir

cualquier archivo que extienda del template, usando block, se puede crear los distintos modulos de codigo que se necesiten

en el template podemos definir un selector y debajo el block

header con la clase x y dentro (identado) block con el nombre

despues del extends e invocar el block el codigo ira dentro de las etiquetas header

Otra funcionalidad de los bloques

un block puede contener valor por defecto y este se aplicara al archivo que lo extienda siempre y cuando no se haya iniciado el block, tendria que comentar el block para que se aplique el codigo por defecto, al agregar codigo en el archivo .pug del cual estamos extendiendo, se perdera el codigo por defecto declarado en el block

para conservar el codigo tendremos que usar append, ej append header, esto traera el codigo por defecto de ese block y permitira crear codigo posterior

si se desea que el codigo por defecto se ejecute antes del codigo posterior, debemos usar prepend

se puede escribir tambien block append header o block prepend header

Includes en pug

se usan para incluir archivos dentro de otros archivos

se define el codigo generalmente en un archivo config dentro de un folder config

generalmente el template base lo incluye, asi include + la ruta, y desde el archivo que extiende de otro hacemos uso de ese codigo

Mixins en Pug

son funciones para reutilizar condigo pug

se inicia con mixin mas un nombre para el mixin, tambien puede recibir parametros

luego se puede incluir, incluso directamente en el index.pug

para usarlo se escribe +nombre del mixin(los parametros que se hayan definido)

+link('/url', 'contacto')
